Rapidly building the game of Crossy Road using AI

A crossy road game built in Python using ChatGPT 5.2 and VS Code.

Jump to overview View final

Overview

This project explores building a playable, Crossy Road–inspired game from scratch using Python and Pygame, with ChatGPT as a development collaborator. Starting from an empty repository, I designed and implemented a minimum viable version of the game, focusing on core mechanics such as grid-based movement, procedural lane generation, obstacle systems, collision detection, scoring, and camera behavior.

The project emphasizes rapid prototyping, iterative debugging, and learning-by-building. Rather than using a prebuilt engine template, I constructed the game’s architecture manually to better understand how fundamental game systems interact.

Project is available on GitHub: https://github.com/ehrsiun/Crossy-Road

Process

I began by defining a clear minimum viable product (MVP): a grid-based character that hops forward through infinite lanes, avoids moving cars, and scores based on forward progress. Establishing this scope helped keep development focused on the core gameplay loop.

I set up a modular project structure separating gameplay logic, world generation, entities, UI, and utilities. This made it easier to build features incrementally and isolate bugs as the project grew.

Next, I implemented tile-based movement and gradually refined it into smooth animated hops with input buffering, allowing players to queue moves for responsive control. I added a simple camera system that follows the player’s forward progress and prevents backward scrolling exploits.

I then built a procedural lane generator that spawns new lanes ahead of the player and removes old ones behind. Road lanes randomly generate cars with varying speeds, sizes, and directions, creating endless variation without hand-authored levels.

Finally, I implemented collision detection, scoring, game states (start, playing, game over), and local score saving. Throughout development, I iterated through debugging environment issues, logic errors, and camera behavior, using small targeted fixes and constant testing to improve stability and feel.

Solution

  • Playable Crossy Road–style MVP with smooth hopping, responsive controls, and infinite procedural lanes
  • Implemented camera, collision, scoring, and game-state systems in Python/Pygame
  • Built with modular architecture for easy iteration and expansion

Skills: Game systems design, procedural generation, Python, debugging, and human–AI collaboration.
