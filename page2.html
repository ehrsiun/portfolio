Live Earth Events Map (NASA EONET)

A lightweight, interactive map that visualizes live natural events using NASA’s EONET API.

Jump to overview View live View code

Overview

This project was created for CMU 15-113 HW3: Explore an API, where the goal is to use AI tools to integrate a public API into a small but interesting app (with interactivity and clear documentation).

I built a single-page web experience that pulls live natural event data from NASA EONET v3 and renders it on a map as clickable markers and polygons. Users can filter by event status, category, and result count, then open an “editorial” side panel for details and sources.

  • Goal: Make an interactive data visualization (not just “fetch and print”) using a real public API.
  • API: NASA EONET v3 (events + categories endpoints, JSON responses with GeoJSON-like geometry).
  • Output: Digital + interactive (map interactions, filters, and a sidebar “story card”).

Process

I started by translating the assignment requirements into a tight MVP: (1) fetch EONET categories, (2) fetch events with parameters, (3) render event geometry on a map, and (4) add interactive controls and a detail view.

Implementation was built iteratively with AI assistance, focusing on getting a stable deployed version first (Vite + GitHub Pages), then layering on interaction and UI polish.

  • Step 1 — API exploration: Read EONET docs, identify the key endpoints (/categories, /events) and filter parameters (status, limit, category).
  • Step 2 — Map MVP: Render the most recent event geometry as markers (Point) and polygons (Polygon), with click-to-select behavior.
  • Step 3 — Interactions: Add hover tooltip/highlight, “Zoom to event,” and filter UI that refetches data when changed.
  • Step 4 — Clarity + packaging: Add a small counts-by-category visualization, write a README explaining the API calls, and create a prompt log for AI collaboration.

Outcome

  • Interactive map dashboard that visualizes live EONET events with hover + click details
  • Filters for status (open/closed/all), category multi-select, and result limits
  • Sidebar “story card” showing category tags, latest geometry timestamp, and source links
  • Compact counts-by-category view to summarize the current dataset
  • Deployed as a static web app (GitHub Pages)

Skills: API integration, data parsing (GeoJSON-like geometry), interactive UI patterns, and rapid prototyping with AI collaboration.
